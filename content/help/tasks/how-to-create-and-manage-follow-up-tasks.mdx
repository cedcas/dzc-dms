## What are tasks?

Tasks are follow-up actions that need to happen by a specific date. They can be linked to a client, a debt account, or be standalone. The Dashboard shows you what tasks are due today and what is overdue.

Tasks are also auto-created when you log a negotiation activity with a next action date and check the "Create follow-up task" option.

## How to create a task manually

1. Go to **Tasks** in the left sidebar (global view) or to the Tasks section on a client or account detail page
2. Click **New Task**
3. Fill in:
   - **Title** (required) — be specific: "Call Capital One re: settlement offer expires 4/15"
   - **Description** — additional context or instructions
   - **Priority** — Low, Medium, High, or Urgent
   - **Due Date** — when this must be done by
   - **Assigned To** — the team member responsible
   - **Client** — link to a client if applicable
   - **Debt Account** — link to a specific account if applicable
4. Click **Create Task**

## Task statuses

| Status | Meaning |
|--------|---------|
| **To Do** | Not yet started |
| **In Progress** | Actively being worked |
| **Done** | Completed |
| **Cancelled** | No longer needed |

## How to complete a task

From the Tasks list or the client/account page:
- Click the **Complete** button (checkmark icon) on the task row — this marks it Done immediately
- Or open the task and update the status to **Done**

## Task priorities

| Priority | When to use |
|----------|-------------|
| **Urgent** | Must be done today — creditor deadline, legal matter |
| **High** | Should be done this week |
| **Medium** | Normal follow-up cadence |
| **Low** | Nice-to-do, no hard deadline |

## Tips

- Set a due date on every task — tasks without due dates will not appear on the Dashboard
- Use Urgent sparingly — if everything is urgent, nothing is
- Assign tasks explicitly — unassigned tasks fall through the cracks
- When a negotiation activity creates a follow-up task automatically, verify the due date and assignee before moving on

## Common mistakes

- Creating a task but not assigning it — it will sit unowned
- Not linking the task to the correct client or debt account — this breaks the task history on account pages
- Completing a task without following through on the action — mark done only when the work is actually done
